# Problem

If notes only accumulate, the knowledge base becomes noisy and less useful over time.

# Claim

Every note should move through a lifecycle so signal stays high and weak notes do not quietly pollute Canonical Knowledge.

# Evidence

Recommended states:

- `inbox`
- `candidate`
- `active`
- `cooling`
- `quarantine`
- `superseded`
- `deleted`

Recommended review cadence:

- weekly light review for new candidates
- monthly review for weakly connected notes
- quarterly review for structure, duplicates, and taxonomy drift

# Applicability

Use this lifecycle for source notes, drafts, canonical notes, and time-sensitive notes.

# Non-goals

- This lifecycle does not require immediate deletion.
- This lifecycle does not replace human judgment for high-value low-frequency notes.
