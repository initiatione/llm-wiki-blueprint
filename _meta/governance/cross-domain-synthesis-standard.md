# Problem

Cross-domain ideas often create duplication, semantic drift, and oversized notes.

# Claim

Cross-domain Synthesis notes should connect domains without replacing their source-of-truth notes.

# Evidence

A good Cross-domain Synthesis note should:

- define the cross-domain problem
- point to the owning domain notes
- describe mappings and limits
- record where transfer works and where it breaks

Avoid:

- copying whole sections from domain notes
- mixing implementation details, theory, and evaluation without boundaries
- turning one synthesis note into a catch-all omnibus page

# Applicability

Use this pattern for notes that truly span multiple domains such as AI and robotics, or programming and research.

# Non-goals

- This is not a shortcut for avoiding domain-specific notes.
- This is not a place to park vague associations.
