# Minimal User Loop

This vault is designed so the user does not need to think about structure most of the time.

## Your Usual Actions

1. Put new material into `_raw/`
2. Let the system turn it into `_drafts/`
3. Review only the cases that actually need your judgment

## What the System Should Decide First

- preferred target folder
- note type
- basic tags and links
- whether the draft looks low, medium, or high risk
- whether the draft should stay in `_drafts/`, move toward canonical folders, or remain conservative

## When You Need to Intervene

- source conflict
- high-risk synthesis
- concept definition that may become a future factual anchor
- architecture or taxonomy change
- deletion, supersession, or strong reinterpretation of existing canonical notes

## Default Philosophy

The user should mostly act as:

- source selector
- final reviewer
- exception handler

Not as:

- manual note organizer
- frontmatter maintainer
- taxonomy operator
