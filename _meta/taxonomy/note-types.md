# Note Types

- `concept`: explains a bounded idea
- `method`: explains a process or technique
- `pattern`: captures repeatable practice or anti-pattern
- `policy`: defines an operating rule or governance constraint
- `reference`: quick lookup note
- `experiment`: records a test, trial, or empirical result
- `decision`: records a choice and rationale
- `project`: stores project-specific knowledge
- `cross-domain-synthesis`: connects multiple domains
- `news`: stores time-sensitive developments
- `source`: records source material and summaries
- `architecture`: documents how the knowledge base itself works
