# Read Order

Use this order unless a task explicitly requires something narrower:

1. `.llm/registry.yaml`
2. `WIKI.md`
3. `wiki/hot.md`
4. `wiki/index.md`
5. Relevant domain or context notes
6. Relevant Cross-domain Synthesis notes
7. `news/` only if recency matters
8. `drafts/` only when reviewing or refining candidate changes

If the task changes architecture, also read:

1. `wiki/architecture/overview.md`
2. `wiki/architecture/invariants.md`
3. Latest ADR in `wiki/architecture/decisions/`
