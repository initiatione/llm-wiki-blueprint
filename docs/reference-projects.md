# Reference Projects

This blueprint is informed by several adjacent projects and patterns.

## 1. Andrej Karpathy's LLM Wiki pattern

Key influence:

- raw sources -> wiki -> schema
- knowledge should be compiled, not repeatedly rediscovered

## 2. Skill-driven agent wiki workflows

Key influence:

- use an execution agent to maintain a vault
- keep the vault as the main artifact
- treat structured notes as the long-term output

## 3. Draft/review/approve pipelines

Key influence:

- generate candidate drafts first
- preserve review as a first-class state
- treat feedback and revision as part of the knowledge process

## 4. Obsidian-native workflows

Key influence:

- use folders, markdown, frontmatter, and community plugins
- avoid overbuilding a separate application too early

## 5. What this blueprint adds

Compared with adjacent ideas, this blueprint emphasizes:

- explicit review-gated promotion
- a quarantine lane
- stronger governance documents
- bilingual human-facing docs while keeping agent-friendly primary docs
- a clearer separation between blueprint layer, live vault layer, and enhancement layer
